FEW-SHOT EXAMPLES FOR CODING AGENTS

Example 1: User shares architecture decision
---
User: "We're using Apache Ignite 3 and prioritizing low-latency reads over write throughput."

Action: Store it IMMEDIATELY
transact:
[[:project/ignite3 :project/ignite3/name "Apache Ignite 3"]
 [:project/ignite3 :project/ignite3/priority "low-latency reads"]
 [:project/ignite3 :project/ignite3/tradeoff "sacrifice write throughput"]]
Reason: Architecture constraint worth remembering across sessions

Response: "I've stored that in memory."

Example 3: Before modifying existing code
---
User: "Add caching to the query layer"

Action: FIRST query memory to check for past decisions
query:
[:find ?e ?a ?v :where [?e ?a ?v]]

Filter results for attributes in the :rules/ namespace to find relevant constraints.

Response: "Based on memory, we have a rule to use Redis for caching."

Example 8: Query historical state
---
User: "What was our roadmap before Phase 2 was added?"

Action: Query as of prior transaction
query:
[:find ?item
 :as-of 5
 :where [?e :project/roadmap/item ?item]]

Response: "At that point, the roadmap had: ..."

---
Example 9: Find items without a property
---
User: "Which components don't have tests?"

Action: Query with negation
query:
[:find ?name
 :where [?e :project/component/name ?name]
 (not [?e :project/component/has-tests "true"])]

Response: "Components without tests: ..."

---
Example 10: Retract obsolete fact
---
User: "Remove the old phase-1 reference - we're in phase-2 now."

Action: First query to find the entity UUID, then retract
query:
[:find ?e :where [?e :project/phase/status "active"]]

Then retract using the UUID returned:
retract:
[#uuid "..." :project/phase/status "active"]
Reason: Outdated - replaced by phase-2

Response: "Removed phase-1 status from memory (preserved in history)."

---
Example 11: Retract and replace with updated value
---
User: "Update the CI monitoring rule description."

Action: Retract old fact, add updated version
retract:
[#uuid "..." :rules/ci-monitoring/description "old description"]
Reason: Replaced with updated description

transact:
[[:rules/ci-monitoring :rules/ci-monitoring/description "updated description"]]
Reason: Updated rule description

Response: "Updated the CI monitoring rule in memory."

Example 12: Track changes over time (valid-time)
---
User: "What was the team's structure in 2025?"

Action: Query with valid-at
query:
[:find ?role
 :valid-at "2025-01-01"
 :where [?e :project/team/role ?role]]
